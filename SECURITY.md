# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not** open a public GitHub issue. Instead, email your findings to: **Wither362**

Please include:

- A description of the vulnerability
- Steps to reproduce it
- The potential impact
- Any suggested fixes (if you have them)

I will acknowledge your report within 48 hours and work with you to address the issue responsibly. Please allow reasonable time for a fix before any public disclosure.

## Security Best Practices

When using this software:

- Keep your projects backed up regularly
- Do not share your project files with untrusted sources
- Report any suspicious activity or unauthorized access attempts
- Use strong passwords if you deploy this to any web service

## Supported Versions

Security updates will be provided for the latest version of this project. Older versions may not receive security patches.

## Disclaimer

This software is provided as-is. While we strive to keep it secure, we cannot guarantee complete protection against all security threats. Use at your own risk.
